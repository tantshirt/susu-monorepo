### What went well (systems-level)

1. **Single curve module** (`calculate_collateral`) reused across post-collateral, top-up, slash, and start gate—fewer divergence bugs later.
2. **Explicit `crate::curve::calculate_collateral` calls** in instructions satisfied the Story 3.1 guardrail and made code search friendly.
3. **IDL freeze + engineering log linkage** preserved auditability when `Group` gained `slash_grace_seconds` and `create_group` gained vault accounts.
4. **Codegen path** remained aligned with **`pnpm sdk:codegen`** and **`scripts/check-sdk-parity.sh`** once generated artifacts stayed in-repo.
5. **Consolidated delivery** validated end-to-end: `anchor build`, `pnpm test:atdd` green in the consolidation pass.

---

### What was hard or surprising (still no blame)

1. **Work landed on local `main` without per-story GitHub PRs** until consolidation—review signals (Bug Bot, human review) ran late relative to commits.
2. **Splitting into “one PR per story” after implementation** proved costly: instructions, IDL, and ATDD assertions are tightly coupled (vault init rewrote assumptions from Story 2.2-era tests).
3. **Rust Anchor lifetime + CPI signer shapes** (`Context<'info, SlashMember<'info>>`, nested seed slices for `transfer_checked`) required focused debugging; this is repeatable teaching material for CPI-heavy stories.
4. **IDL source of truth drift**: committed `programs/susu/idl/susu.json` can lag **`target/idl`** after `anchor build`—teams need one chosen sync rule in the workflow checklist.
5. **`slash_member` remaining-accounts participation model** trusts clients to pair honest `(MemberPosition, ATA)` pairs; security and ergonomics merit a future hardening story if not already specified.
