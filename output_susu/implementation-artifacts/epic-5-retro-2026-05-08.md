## What Went Well

- The epic produced a coherent evidence chain: `tests/invariants/no_strategic_default.rs`, `crates/susu-adversary`, `audits/adversary/adversary-report.json`, `docs/collateral-curve.md`, and `docs/threat-model.md` now reinforce the same Curve Invariant claim.
- Dependency order worked: 5.2 enabled 5.3 and 5.4; 5.1 and 5.4 enabled 5.5; 5.7 enabled 5.9.
- The deterministic report work tightened the simulator boundary by removing nondeterministic metadata paths and adding canonical byte comparison.
- Documentation stories were not treated as prose-only work. They added scripts, coverage matrices, artifact indexes, and path-existence checks.
- Cursor findings were useful when terminal, especially the 5.4 build-script cleanup and 5.2 build-script/SDK dependency cleanup.
- PR #173 made the Story 5.3 exception auditable instead of leaving the sprint tracker silently inconsistent.

## Challenges

- Story 5.3 violated the normal merge discipline: PR #172 merged after CI passed while Cursor Bugbot remained queued/ambiguous. That required an explicit user-approved manual override and follow-up documentation.
- Artifact drift persisted after merge. Story 5.2 still said `Status: review` even though sprint status, PR #170, and the issue state showed done.
- Several Epic 5 claims depend on careful wording around evidence strength. Static checks, deterministic model tests, and full live replay are not interchangeable.
- The adversary report has a self-referential seed/reproduction constraint. The repo now documents the constraint, but future release automation must preserve the exact convention.
- Documentation and workflow stories can look complete before their external dependencies are final. The audit and legal workflows must avoid implying that third-party conclusions already exist.

## Key Lessons

1. A manual override can be valid, but it must be named in the artifact trail with the exact reason, PR number, and residual risk.
2. Status authority needs a single reconciliation step after merges. Story files, sprint status, dependency graph, PR state, and issue state must not drift.
3. Determinism is a whole-system property. RNG seeding, report ordering, metadata, build scripts, and CI reproduction all matter.
4. Public claims need verifier paths. The strongest Epic 5 artifacts cite concrete source, report, script, or matrix files rather than relying on narrative.
5. Compliance documentation must describe structural posture and re-review triggers without pretending to be a legal opinion.
6. External-facing evidence should separate "implemented", "verified by repo checks", "ready for auditor review", and "externally accepted".
