### Phase 2 ‚Äî Epic 9 scaffolding (Stories 9.2 / 9.3 / 9.4)

Mainnet ceremony itself stays gated on audit completion (`audits/SKIP_AUDIT_GATE` is still committed). This work ships the **engineering scaffolding** that lets the operator run the ceremony in one command once the audit lands.

**Story 9.2 ‚Äî `scripts/deploy-mainnet.sh`** (new):

- Atomic deploy + burn ‚Äî `solana program deploy --upgrade-authority 1nc1nerator11111111111111111111111111111111` ‚Äî so there's no human-error window between deploy and burn.
- Flags: `--cluster {mainnet-beta|devnet}`, `--dry-run`, `--program-keypair`, `--payer`, `--rpc-url`, `--skip-audit-gate` (devnet only, refused against mainnet-beta).
- Pre-flight: refuses to run against mainnet-beta if `audits/SKIP_AUDIT_GATE` exists; otherwise calls `scripts/check-audit-signoff.sh` (Story 9.1 gate).
- On success, writes `MAINNET_PROGRAM_ID.md` (or `DEVNET_PROGRAM_ID.md`) at the repo root with program ID, deploy timestamp, deploy SHA, IDL SHA-256, and cluster ‚Äî the artifact that arms Story 9.3 and 9.4.
- Devnet dry-run `bash scripts/deploy-mainnet.sh --cluster devnet --dry-run` exits 0 cleanly with the expected `solana program deploy` command printed.

**Story 9.3 ‚Äî `.github/workflows/immutability-check.yml`** (new):

- Triggers on push to `main` + nightly cron (09:00 UTC) + `workflow_dispatch`.
- Skips if `MAINNET_PROGRAM_ID.md` doesn't exist (pre-mainnet no-op ‚Äî keeps the workflow harmless until 9.2 fires).
- Parses program id from the `MAINNET_PROGRAM_ID.md` table row, installs Solana CLI v2.0.21, runs `bash scripts/check-immutability.sh` with `CLUSTER=mainnet-beta SUSU_PROGRAM_ID=<id>`.
- Smoke tested both modes: `CLUSTER=devnet` ‚Üí "skipped (mainnet-only)" exit 0; `CLUSTER=mainnet-beta` with no real deploy ‚Üí expected RPC `Unable to find the account` failure (correct protective behavior).

**Story 9.4 ‚Äî badge wiring** (incremental):

- New helper `apps/reference/lib/badge/load-mainnet-program-id.ts` ‚Äî defensive parse of the `| Program ID | \`<id>\` |` row from `MAINNET_PROGRAM_ID.md`, returns `undefined` if file missing or row malformed (degrades the badge to `pending` rather than crashing).
- `apps/reference/app/api/badge/upgrade-burned/route.ts` resolution order updated to: explicit test override ‚Üí file ‚Üí env. Once the operator commits `MAINNET_PROGRAM_ID.md`, the badge auto-flips on the next build.
